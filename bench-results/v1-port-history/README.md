# bench-results/v1-port-history/

Empirical artifacts ported forward from `v1-maintenance` during the diarize-stack
v1→v2 port (iter 215, `feat/v2-diarize-port` → `feat/v2-thin-client`).

These files are NOT regenerated by v2 bench targets — they are the v1
investigation trail kept on v2 so the reasoning behind the diarize-stack
defaults (collapse_threshold = 0.65, apply_collapse semantics, ghost-cluster
filtering) remains discoverable. v2-side benches that produce new artifacts
should write under `bench-results/phase-X/` (without the `v1-port-history/`
prefix) so the audit trail stays separated.

## Reading order

1. `phase-a/analysis.md` — diagnoses the segmentation-model overcount;
   surveys threshold candidates against the four centroid dumps.
2. `phase-d/SUMMARY.md` (or equivalent narrative) — confirms 0.65 meets
   the headline UX target on the canonical debate fixture.
3. `phase-d/*.json` — bench-raw outputs cited by the summary.

## Why kept on v2

Dropping these would lose the reasoning trail future threshold-tuning work
needs. Renaming them under `v1-port-history/` (vs. a bare `phase-a/`
sibling to future v2-side bench dirs) keeps v2's tree v2-clean: the
`v1-port-history/` prefix unambiguously marks them as "investigation
inherited from v1, not v2-produced".
